剛剛在無聊的情況下 測試wifi干擾  
**全程內網 請不要在家裡之外的可控範圍實作 不然會觸犯刑法**  

設備:  
紅磨平版3pro(國際)  
oneplus 15 (ROOT)  
三星 M14 (ROOT)  
使用termux   
使用模組  
{  
sudo nmap hping3  
}  

在內網測試的情況下  
在沒連接路由器(平板[平板只有2.4ghz wifi6 and wifi7 兩者干擾力度差不多])  
被攻擊設備手機連wifi(S24 FE  連接網路路由器[不是平板])  
從500Mbps 掉到99Mbps 設定正確的情況下 只有40  
但是如果路由器支援兩者頻段 5G/2.4G  
則從500掉到300~200  
兩者的上傳分別是374 -> 200 設定正確的情況下 60~20  
  
4G LTE 基本上只有10%左右的影響  
從83->75(下載) 10.6->3.28(上傳[不知道是不是誤差])  
  
平板只有5G (所以上述攻擊是在5Ghz上發生的 沒測試專門針對2.4Ghz的影響)  
但是另一個情況下  
攻擊設備連接路由器 內網攻擊  
A分享路由 / B分享路由 (wifi分享路由器 這時候平板沒介入了)  
  
在B分享路由兩個設備互相攻擊 不容易影響A路由  
500->200~300 上傳400->260~300  
但在B路由測速  
直接變成20~60(兩者)  
如果設定正確 就無法連線(可能分享路由當機了)  
  
然後用2.4G跟5G一起攻擊 效果差不多  
(oneplus 支援雙wifi 主連5Ghz 副連2.4Ghz 另一隻攻擊2.4Ghz oneplus 挑手機透過5Ghz攻擊)  
然後路由器炸了
